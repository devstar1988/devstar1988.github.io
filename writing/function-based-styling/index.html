Function-based Styling

My transition from component-based to function-based styling and how it sparked an interest in design systems.

When I started learning about front-end development my CSS files were a giant knot of overly specific classes that were impossible to manage. The bigger the stylesheet the bigger the knot. My stylesheets were such a tangled mess that making a change to an existing class created an unintentional ripple effect throughout my sites. As a consequence, to make a predictable change I had to create a new, more specific class. Which meant a bigger stylesheet. Which resulted in a bigger knot.

Highly specific classes resulted in cluttered, unmanageable stylesheets

I was frustrated, so I started questioning and looking for solutions. How do the best developers and top product companies manage their stylesheets? I chatted with designer friends, inspected websites, read articles, and re-listened to several Design Details episodes. A friend of mine shared CSS and Scalability, an article written by Adam Morse that talks about the benefits of function-based styling and his experience managing stylesheets. After reading more of Adam’s thoughts on CSS and learning about his active involvement with the development community, I found Tachyons, one of his side projects, and decided to give it a try on my next client project.

At first, function-based styling felt wrong because it required an opposite approach to component-based styling. Instead of building UI components by adding a single, multi-property class I was building components by adding multiple, single-property classes. My component-based mindset refused to share a button’s padding with the site’s navigation menu and it hated cluttering my markup with so many classes. It wasn’t until I completed the first webpage that I became aware of the power of function-based styling. I realized my markup was readable, my classes reusable, and changes were predictable. What was once a bloated knot of overly specific classes became a collection of lean, globally reusable properties.

Isolated global classes resulted in manageable and scalable stylesheets

After working with Tachyons on my first client project, I started building websites faster and creating lean, manageable, and scaleable stylesheets. I quickly fell in love with the system mindset that is a result of prioritizing reusability. Instead of creating random, unrelated values for a component I started creating specific, scalable values for the entire website. Learning about function-based styling sparked an interest in learning more about design systems and their function in scaling digital products.
